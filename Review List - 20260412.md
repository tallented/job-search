# Review List - 20260412

Last updated: 2026-04-12

This queue refreshes the search using the three-lane strategy in `Search Strategy - 20260412 - Executive Scale.md`.

Important update for future passes: `application / product engineering leadership` is now the default preference. `Platform / DevEx / Cloud` remains a valid bridge lane, while `security / corporate engineering` should be treated as opportunistic rather than default.

Priority order for this pass:

1. `Lane 1 + Lane 2 overlap`: executive bridge roles at strong brands, especially with product/application scope
2. `Lane 1`: direct bridge roles toward larger-company `CTO` / `EVP Technology`, with application/product engineering preferred
3. `Lane 2`: top-tier or strong-brand management roles, especially over customer-facing software teams
4. strong `platform / DevEx / cloud` bridge roles that stay close to product delivery
5. `security / corporate engineering` only when scope, brand, or compensation are unusually strong
6. `Lane 3`: selective principal / distinguished upside only

This pass is intentionally biased toward:

- remote or remote-friendly work
- application / product engineering leadership
- customer-facing software and workflow-heavy app development
- greenfield builds or major application modernization
- roles that increase scale credibility
- platform / governance / AI / developer productivity themes when they strengthen the app-engineering path
- compensation at or near the current practical floor unless the brand leverage is unusually strong
